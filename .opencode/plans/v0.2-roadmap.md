## v0.2 Plan

### A. GUI Static Views — Real Backend (1-2 days)
Wire Dashboard, ProfileSelector, Revert to live PsRunner calls.
- Dashboard: `Audit -Profile X -OutputJson` → parse score, timestamp
- ProfileSelector: `List -OutputJson` → populate from parsed JSON
- Revert: scan `%LOCALAPPDATA%\WinTune\backups\*` for sessions

### B. GUI Live Apply Progress — Design Decision (2-4 days, blocked on decision)
Three options:
1. **Polling:** engine writes progress JSON to `%TEMP%\wintune-progress\<session>.json` each step. GUI polls.
2. **Streaming:** engine writes structured lines to stdout during execution. GUI reads with `StreamReader.ReadLineAsync()`.
3. **Status quo:** spinner + "Applying..." until exit, show final result.

**Decision needed before implementation.**

### C. Exit Code 2 — Partial Success (0.5 day)
Frozen in cli-spec.md but not implemented. Exit `2` when changes contain failures without `-StopOnError`.

### D. Build-Aware Scoring in Get-Score (1 day)
Filter tweaks by `buildMin`/`buildMax` before scoring. Prerequisite for windows10 profile. Currently Get-Score counts build-inapplicable tweaks in Total, inflating score on older builds.

### E. Tweak Database Expansion — By Category (2-3 days)
Target categories, not numbers:
- **Retail packages:** BingNews, BingWeather, GetHelp, OfficeHub
- **Telemetry services:** dmwappushsvc, WpnService
- **Xbox services:** XblAuthManager, XblGameSave, XboxGipSvc, XboxNetApiSvc (toggleable per profile)
- **Edge & web:** registry keys for Edge preload/startup boost, Edge scheduled tasks
- **Scheduled tasks:** Office telemetry, CEIP, Application Experience

Hard constraint: every entry must have a detect script. No entry without testable detection.

### F. Command-Type Revert (0.5 day)
Restore-Backup throws for `command` type. Implement undo mapping:
- `disable-hibernation` → `powercfg /h on`
- Generic `$commandUndo` hashtable for future commands

### G. Scheduled Audit (1 day, blocked on admin policy decision)
Register weekly ScheduleTask that runs Audit and writes `logs\audit\<YYYY-MM-DD>.json`.
Open question: Audit requires admin. Run task as SYSTEM, or add admin-free reduced-scope audit?

### H. Tech Debt (1-2 days)
- **Helpers.ps1:** move `Test-CommandDetected` out of TweaksEngine → `src/modules/Helpers.ps1`
- **CI Rust build:** add `cargo build` for gui-tauri to ci.yml
- **OutputJson contract tests:** Apply/Revert shapes not covered (blocked by admin requirement)

## Priority Order
```
A (GUI static) ── no dependencies ── start now
D (build scoring) ── blocks E (tweak expansion)
C (exit code 2) ── no dependencies
F (command revert) ── no dependencies
B (live progress) ── needs design decision
G (scheduled audit) ── needs admin policy decision
H (tech debt) ── background
```

Recommended: A → C → D → F → E → (decide B) → (decide G) → H

## Questions to Resolve

1. **Live Apply progress — Option 1, 2, or 3?** Streaming (Option 2) is cleanest but requires engine changes. Polling (Option 1) is simpler to implement but feels hacky. Status quo (Option 3) is zero-cost but less polished.

2. **Scheduled Audit — run as SYSTEM or add admin-free Audit light?** Current Audit requires admin. Options: add `-QuickScan` that skips services (no admin needed for packages + tasks + registry), or run the task as SYSTEM via schtasks.

3. **GUI ViewModels — refactor to async or keep sync with Task.Run?** Currently ViewModels are simple POCOs. Wiring PsRunner (async) requires async ViewModels with loading states, error handling, cancellation support.

4. **Tweak database categories — which to prioritize?** Telemetry services have highest impact (privacy + performance). Edge background tasks are noisy on fresh installs. Xbox services are profile-dependent (gaming keeps them, everyone else removes).
